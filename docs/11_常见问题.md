# 常见问题和排查顺序

## 无法烧录

1. 检查 PA13/PA14 是否接反、GND 是否共地。
2. 确认目标板有 3.3 V，ST-Link 检测到目标电压。
3. 降低 SWD 速度到 1000 kHz 或更低。
4. 检查 BOOT0 应为低电平。
5. 必要时按住复位再连接，但不要随意解除读保护或全片擦除用户数据。

## 烧录后没有任何输出

- 工程默认要求 8 MHz HSE；晶振规格不同会导致时钟配置失败。
- USART1 是 PA9/PA10、115200、8-N-1。
- PC13 LED 为低电平亮，错误状态会闪烁。
- 输入 `VERSION`；如果没有回复，先用 USB 转串口替代 HC-05 排除蓝牙配置问题。

## 一接舵机或电机就复位

这是典型电源问题。舵机不能由 3.3 V 供电，电机和舵机的浪涌也不能经过主控板小稳压器。
使用独立 5 V DC-DC、VM 大电容、短粗电源线并共地。用 `DIAG` 查看 reset 位 0x02 常见于
电源复位，0x08 表示 IWDG。

## 电机不转

- `STATUS` 查看 fault 是否非零。
- 确认 TB6612 VM、VCC、STBY 和共地。
- 发送 `CLEAR` 后持续发送 `DRIVE 30 30`，否则 1 秒超时。
- 如果 fault 包含 0x20，表示堵转锁存；先排除机械/编码器问题再 CLEAR。

## PID 输出异常或立即满速

先检查编码器符号。正向行驶时 RPM 必须为正；符号相反会形成正反馈。再检查 CPR 是否包含
AB 相四倍频。只有方向和比例正确后才调整 PID。

## MOVE 距离不准

- 直线距离成比例偏差：修正轮径或 CPR。
- 左右偏航：两轮轮径、机械阻力或 PID 不一致。
- 地面打滑：降低 SPEED 和 RAMP。
- 不要使用轮胎标称直径，应该测量承重后的有效滚动直径。

## TURN 角度不准

先保证左右轮距离基本准确，再通过有效轮距修正。有效轮距不是简单的两轮中心物理距离，
它会受轮胎侧滑影响，应通过 `TURN 360` 实测标定。

## ODOM 漂移

纯编码器里程计必然累计误差，转弯和打滑时更明显。本工程适合短路径和可重复动作，不适合
长时间绝对定位。需要更高精度时应增加 IMU 或外部定位。

## HC-SR04 总是显示 400 cm

400 cm 是超时/无有效回波的占位值。检查 5 V、共地、Trig、Echo 分压和 PA4。软布、斜面或
超出量程也可能无回波。

## OLED 不亮

确认是 SSD1306 128×64、地址 0x3C、PB8/PB9 接线和 3.3 V 供电。工程软件 I2C 不会等待
ACK，因此 OLED 缺失不会阻塞系统，可通过串口继续诊断。

## SAVE 后参数没有保留

1. 车辆必须完全停止，否则返回 `ERR STOP FIRST`。
2. 应看到 `OK SAVED`。
3. 用 `CONFIG` 查看 sequence 是否增加。
4. 确认使用增强版链接脚本；其他工程如果写满 64 KB，可能覆盖参数页。

## DIAG 中栈高水位很小

单位是 32 位字而不是字节。先运行所有功能后再判断。如果某任务接近零，减少该任务的大型
局部数组或格式化字符串；确认后才增加栈，并重新检查 C8T6 FreeRTOS 堆余量。
