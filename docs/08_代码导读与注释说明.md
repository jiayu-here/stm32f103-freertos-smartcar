# 代码导读与修改入口

## 1. 目录结构

```text
Core/Inc             HAL 和 FreeRTOS 配置
Core/Src             main、中断、MSP、最小系统调用
App/Inc              车辆状态、PID、协议、参数结构接口
App/Src              七任务、模式逻辑、PID、协议、Flash 参数
BSP/Inc              板级引脚和驱动接口
BSP/Src              时钟、GPIO、TIM、ADC、UART、电机、OLED
Drivers              STM32CubeF1 HAL 和 CMSIS
Middlewares          FreeRTOS 内核、heap_4、Cortex-M3 移植层
startup              C8T6/RCT6 启动文件
linker               程序区与参数页边界
scripts              构建、烧录、测试、串口控制台
artifacts            预编译 ELF/HEX/BIN/MAP
```

## 2. 启动调用链

```text
Reset_Handler
  -> SystemInit
  -> main
     -> BSP_Init                 时钟和外设初始化
     -> App_Init                 加载参数、创建 RTOS 对象
     -> BSP_StartPeripherals     启动 PWM/编码器/UART/IWDG/OLED
     -> App_StartScheduler       创建七任务并启动调度器
```

## 3. 关键文件

### `App/Src/app.c`

系统核心。包含：

- ControlTask：编码器、RPM、里程计、速度斜坡、PID、堵转。
- SafetyTask：低电、按键急停、命令超时、堵转、最终电机授权。
- CommunicationTask：接收流、命令解析和响应。
- BehaviorTask：五种模式、定距定角和路线。
- SensorTask：红外和电池采样。
- DisplayTask：OLED 状态/里程计页面。
- MonitorTask：健康位、看门狗、状态日志。

设计原则是 BehaviorTask 只生成左右 RPM 目标，不直接操作 PWM；ControlTask 只在
SafetyTask 授权后驱动电机。

### `App/Src/app_config.c`

定义默认标定值、参数范围、CRC32、Flash 扫描、追加写入和读回校验。修改结构体时必须：

1. 增加 `APP_CONFIG_VERSION`；
2. 更新默认值和范围检查；
3. 保证结构长度是 4 字节倍数；
4. 重新做 C8T6 链接和 Flash 保存测试。

### `App/Src/protocol.c`

纯 ASCII 命令解析器。它会把输入转为大写，填充 `ParsedCommand`，不直接操作硬件，因此
可以在 Windows 主机测试。新增命令时同步修改 `protocol.h`、解析器、`execute_command()`、
帮助文字和 `tests/test_pid_protocol.c`。

### `BSP/Src/bsp.c`

板级驱动和外设初始化。更换引脚、时钟、PWM 频率、UART 波特率、ADC 通道或有效电平时
在此修改。应用层不应该直接访问 GPIO/TIM 寄存器。

### `BSP/Inc/board.h`

唯一的固定引脚定义。左右编码器方向宏仍在此处；会掉电保存的标定参数位于
`app_config.c`，避免出现两套相互矛盾的默认值。

### `Core/Inc/FreeRTOSConfig.h`

C8T6 使用 10 KB RTOS 堆，RCT6 使用 20 KB。中断优先级规则：调用 FreeRTOS ISR API 的
中断数值优先级必须不高于允许边界，本工程 USART1/EXTI4 使用优先级 6。

## 4. 共享数据和临界区

`CarState` 和 `AppConfig` 会被多个任务访问。短结构复制与字段更新使用 FreeRTOS 临界区，
禁止在临界区内执行 UART、Flash 擦除、OLED刷新或延时。UART 发送用互斥量；USART 接收
和超声完成使用 StreamBuffer/信号量从 ISR 通知任务。

## 5. 如何增加功能

新增传感器通常按以下结构：

1. 在 `board.h` 分配无冲突引脚；
2. 在 BSP 创建初始化和读取接口；
3. 在低优先级传感器任务采集；
4. 把结果写入 `CarState`；
5. 行为任务使用数据生成速度目标；
6. 安全相关判断放入 SafetyTask；
7. 增加协议、OLED、文档和验收用例。

不要在中断里做浮点运算、串口格式化、Flash写入或长延时。

## 6. 注释约定

代码注释重点解释“为什么”：参数页为什么预留、航向为什么不折返、堵转为什么要延时、
安全故障为什么清除目标。明显的赋值和 HAL 调用不重复逐行注释，以避免注释和代码脱节。
