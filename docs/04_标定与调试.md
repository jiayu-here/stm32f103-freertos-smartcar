# 参数标定与调试方法

## 总原则

先校正方向和比例，再调控制参数。错误的编码器符号无法靠 PID 修正。所有初次电机测试都应
架空车轮，并准备随时按下急停。

## 1. 电机方向

持续发送 `DRIVE 30 30`，两轮应向前。若某轮反转，优先交换该电机两根输出线。这样源码中
“正 RPM 代表前进”的约定保持一致。

## 2. 编码器方向与 CPR

正向转动车轮时 STATUS 的对应 RPM 必须为正。若为负，修改 `board.h` 的编码器 sign。

CPR 应使用定时器四倍频后的机械轮一圈计数。可在电机断电时手动缓慢转一圈，用调试器查看
定时器计数差，或临时增加原始计数日志。齿轮箱电机应按最终输出轴而不是电机轴计算。

设置示例：

```text
WHEEL 6.50 14.50 780
```

## 3. 轮径

测量轮胎承重后的有效直径。先输入近似值，执行 `MOVE 100` 并测量真实距离：

```text
新轮径 = 当前轮径 × 真实距离 / ODOM 报告距离
```

重复 2～3 次，使用较低速度并避免打滑。

## 4. 有效轮距

先完成直线距离标定，再执行 `TURN 360`。如果真实角度小于里程计角度，应增加还是减小轮距
可按公式直接修正：

```text
新轮距 = 当前轮距 × ODOM 报告角度 / 真实角度
```

轮胎侧滑使“有效轮距”和物理中心距不同，这是正常现象。

## 5. PID

架空车轮，关闭积分和微分：

```text
PID 0.30 0 0
RAMP 120
DRIVE 50 50
```

1. 增加 Kp，直到响应足够快但没有持续振荡。
2. 缓慢增加 Ki，消除稳态误差。
3. 只有噪声和超调确实需要时才加入少量 Kd。
4. 测试 30、60、100 RPM 以及前进、后退和差速。
5. 落地后再次微调，因为负载与架空不同。

确认后输入 `SAVE`。调参中不要频繁 SAVE，避免无意义 Flash 写入。

## 6. 加速度斜坡

默认 180 RPM/s。小车起步打滑或电源跌落时减小；响应过慢时增加：

```text
RAMP 120
```

安全停车不受斜坡限制，故障会立即关闭电机。

## 7. 循迹增益

中心探头压线时 IR 应主要为 `04`。线路偏左时低位探头变化，偏右时高位变化。基础速度先设
40～60 RPM：

```text
SPEED 50
LINEGAIN 10
MODE TRACK
```

转弯不足则增加 LINEGAIN，左右摆动则减小或降低 SPEED。

## 8. 超声波和舵机

用平面硬障碍物在 10、30、100 cm 处比较 STATUS 距离。400 cm 代表超时或无有效回波。
确认舵机 -55° 对应车体左侧、+55° 对应右侧；若相反，可调整机械安装或扫描角度映射。

## 9. 电池

用万用表测电池真实电压，并读取 STATUS：

```text
新倍率 = 当前倍率 × 万用表电压 / STATUS 电压
```

例如：

```text
BATTERY 6.60 6.90 4.0410
SAVE
```

低电测试使用可调电源，不要故意把锂电池深度放电。

## 10. 堵转参数

默认 `STALL 75 4 1000`，表示 PWM 至少 75%、速度低于 4 RPM 持续 1000 ms。阈值过低会在
正常慢启动时误报，过高则保护太迟。测试时架空车轮，短时间阻止编码器反馈，并监视电机与
驱动温度。堵转故障为 0x20，排除原因后发送 CLEAR。

## 11. 诊断

覆盖全部模式后执行 `DIAG`。重点查看：

- `heap_min` 不能接近 0；
- 任一 `stack_words` 不应接近 0；
- reset=0x08 表示上一次为 IWDG 复位；
- `LOG` 中反复出现相同故障说明阈值或硬件有问题。

标定完成后执行 `CONFIG` 保存一份文本记录，便于之后恢复。
