# 项目总览与实现功能

## 1. 项目目标

本项目以 STM32F103C8T6 为最低资源目标，在 FreeRTOS 上实现一套两轮差速智能小车。
设计重点不仅是“能跑”，还包括实时控制、任务隔离、故障保护、在线标定、参数保存、
运行诊断和可以重复验收的工程流程。

STM32F103RCT6 使用相同业务源码，只更换器件宏、启动文件、链接脚本和 RTOS 堆大小。

## 2. 完整功能清单

### 2.1 电机与速度控制

- TIM2_CH1/CH2 输出 20 kHz PWM，避免可闻频率噪声。
- TB6612FNG 控制两路直流减速电机的正转、反转、停止和待机。
- TIM3/TIM4 使用硬件编码器模式读取两轮 AB 相编码器。
- 控制任务每 10 ms 计算轮速并执行左右轮独立 PID。
- 对目标 RPM 加入可调加速度斜坡，避免指令阶跃造成打滑和电源跌落。
- 当 PWM 长时间较高但编码器速度接近零时锁存堵转故障。

### 2.2 五种基础模式

| 模式 | 作用 |
|---|---|
| MANUAL | 蓝牙或串口直接设置左右轮 RPM，1 秒无新运动指令则停车 |
| TRACK | 五路红外加权循迹，丢线时按最后偏差方向搜索 |
| AVOID | 超声波检测前方障碍，舵机左右扫描后选择路径 |
| FUSION | 正常循迹，遇到近距离障碍后执行扫描避障 |
| CRUISE | 按基础速度直行，前方过近自动停车 |

内部还包含 `MOTION` 状态，用于定距、定角和路线队列，不通过 `MODE` 命令直接选择。

### 2.3 定量运动与路线

- 通过轮径、轮距和编码器 CPR 计算左右轮距离。
- 根据差速模型累计 X、Y 和不折返的航向角。
- `MOVE 100`：前进 100 cm；负值表示后退。
- `TURN 90`：原地左转 90°；负值表示右转。
- 最多保存 8 个 RAM 路线段，使用 `QMOVE/QTURN` 组装后由 `RUN` 连续执行。
- 每个运动段末端会自动降速，完成后输出事件消息。

里程计会受到轮胎打滑、轮径误差和轮距误差影响，不等同于高精度定位系统。

### 2.4 传感器与交互

- 五路红外数字量循迹。
- HC-SR04 EXTI 双边沿回波计时。
- SG90 50 Hz 舵机扫描。
- SSD1306 OLED 在运行状态页和里程计页之间自动切换。
- HC-05/USART1 支持模式、速度、PID、标定、路线、诊断和状态命令。
- 电池 ADC 采样与低电迟滞保护。

### 2.5 安全和诊断

- PA15 物理急停，按下后锁存，必须松开并发送 `CLEAR`。
- `STOP` 软件急停同样锁存。
- 手动遥控 1 秒超时保护。
- 低电保护使用低阈值和恢复阈值，避免临界电压反复启停。
- 堵转锁存保护，`CLEAR` 后才允许再次运行。
- FreeRTOS 栈溢出与堆分配失败钩子。
- 所有关键任务必须周期性上报健康位，监控任务满足条件才喂 IWDG。
- `DIAG` 查看剩余堆、历史最小堆、各任务最小剩余栈和复位原因。
- `LOG` 查看最近 8 次故障状态变化；日志位于 RAM，复位后清空。

### 2.6 参数掉电保存

可保存 PID、基础速度、加速度、循迹增益、轮径、轮距、CPR、电池参数和堵转参数。
参数区使用最后一页 Flash，应用程序链接区域不会覆盖它。每次 `SAVE` 追加一条带 CRC 的
记录，页面写满后才擦除，从而减少 Flash 擦写次数。

## 3. 未加入的功能

为了保持现有硬件和 C8T6 资源边界，本版没有加入：

- IMU 航向融合和磁力计
- 电流传感器的真实过流测量
- 摄像头、视觉识别和 SLAM
- 蓝牙固件升级 Bootloader
- 手机 App 源码

这些功能需要新硬件、额外引脚或更大的验证范围，适合作为后续 RCT6 扩展。

## 4. 完成和验收边界

代码已经通过 C8T6/RCT6 交叉编译和主机协议/PID测试。没有实车时无法替代以下测量：
电机方向、编码器符号、真实 CPR、轮胎直径、有效轮距、舵机方向、红外有效电平、PID 和
电池分压误差。使用者必须完成 `docs/05_验收清单.md`。
