# 从零到运行操作步骤

## 阶段 A：准备

1. 按 `hardware/BOM.csv` 准备组件。
2. Windows 安装 Arm GNU Toolchain 14.x、ST-Link 驱动和 OpenOCD 或 STM32CubeProgrammer。
3. 在工程根目录运行：

```powershell
.\scripts\install_tools.ps1
.\scripts\test_host.ps1
.\scripts\build.ps1 -Device C8 -Clean
```

出现 `BUILD OK` 后，在 `build/C8` 得到 ELF、HEX、BIN、MAP。

## 阶段 B：最小系统烧录

只连接 STM32、ST-Link 和稳定电源：

| ST-Link | STM32 |
|---|---|
| SWDIO | PA13 |
| SWCLK | PA14 |
| GND | GND |
| 3.3 V reference | 3.3 V |

运行：

```powershell
.\scripts\flash.ps1 -Device C8
```

脚本会执行写入、verify 和复位。失败时不要继续接电机，先排查供电和 SWD。

## 阶段 C：低功率外设

依次连接 OLED、HC-05、红外和 HC-SR04。HC-SR04 Echo 必须经 10k/20k 分压。打开串口：

```powershell
python .\scripts\car_console.py COM5
```

输入：

```text
VERSION
STATUS
CONFIG
DIAG
```

应看到版本、状态、默认参数、堆栈和复位原因。OLED 应在状态页和里程计页间切换。

## 阶段 D：电机和编码器

1. 舵机暂不接，架空车轮。
2. TB6612 VM 接电机电源，VCC 接 3.3 V，所有地共地。
3. 发送 `CLEAR`，再持续每 500 ms 发送 `DRIVE 30 30`。
4. 两轮应向前，状态中的左右 RPM 应为正。
5. 如果电机方向错，交换电机线；如果 RPM 符号错，修改 `LEFT/RIGHT_ENCODER_SIGN`。
6. 实测机械轮旋转一圈的计数，设置：

```text
WHEEL 6.50 14.50 780
SAVE
```

上例依次是轮径 cm、有效轮距 cm、编码器四倍频每圈计数，必须换成实测值。

## 阶段 E：PID 和斜坡

从低速度开始：

```text
PID 0.30 0 0
RAMP 120
DRIVE 50 50
```

逐步调整 Kp、Ki、Kd。每次测试期间持续发送 DRIVE，避免 1 秒手动超时。确认后：

```text
PID 0.60 0.35 0.005
RAMP 180
MODE MANUAL
SAVE
```

## 阶段 F：里程计

在地面画出 100 cm 直线，执行：

```text
RESETODOM
MOVE 100
ODOM
```

距离偏差优先修正轮径/CPR。再执行 `TURN 360`，旋转角偏差主要通过有效轮距修正。

## 阶段 G：自动功能

```text
MODE TRACK
MODE AVOID
MODE FUSION
MODE CRUISE
```

逐个模式低速测试，不要第一次就使用高速度。自动模式基础速度由 `SPEED` 设置。

## 阶段 H：路线队列

```text
CLEARROUTE
QMOVE 100
QTURN 90
QMOVE 50
QTURN -90
ROUTE
RUN
```

运行中可发送 `CANCEL` 平稳取消，紧急情况发送 `STOP` 或按下 PA15。

## 阶段 I：保护验收

1. 手动模式停止发送 DRIVE，确认约 1 秒停车。
2. 按急停，确认电机立即停止；松开后发送 CLEAR 才能恢复。
3. 用可调电源验证低电阈值，禁止通过深度放电锂电池测试。
4. 架空车轮并人为阻止编码器反馈，验证堵转锁存；测试时防止电机过热。
5. 输入 `LOG`、`DIAG` 检查故障历史和栈堆余量。
