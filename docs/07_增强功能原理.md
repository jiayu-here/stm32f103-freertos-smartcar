# 增强功能实现原理

## 1. Flash 参数保存

### 地址分配

| 芯片 | 程序链接区域 | 参数页 |
|---|---|---|
| C8T6 | 0x08000000 起，63 KB | 0x0800FC00，1 KB |
| RCT6 | 0x08000000 起，254 KB | 0x0803F800，2 KB |

链接脚本主动缩小程序 Flash，因此即使固件继续增长，链接器也会在覆盖参数页前报错。

### 记录格式

每条记录包含：魔数、格式版本、结构长度、递增序号、全部参数和 CRC32。启动时扫描参数页，
选择序号最大的合法记录。掉电导致的半写记录 CRC 不正确，会被自动忽略。

`SAVE` 不会每次擦除页面，而是写入下一个空槽。页面没有空槽时才擦除整页。这是简单的
追加式磨损均衡。只有车辆完全停止时才允许 `SAVE/LOAD/DEFAULTS`。

## 2. 目标速度斜坡

串口或行为任务给出最终目标 RPM，控制任务内部维护一个“实际用于 PID 的斜坡目标”：

```text
每周期最大变化 = acceleration_rpm_s × 0.010 s
```

默认 180 RPM/s，因此从 0 到 90 RPM 约需 0.5 秒。安全故障不等待斜坡，直接停止 PWM 和
TB6612 STBY，保证保护优先级高于舒适性。

## 3. 堵转检测

堵转判定同时满足以下条件：

1. PID 输出绝对值达到配置的 PWM 阈值；
2. 斜坡目标绝对值至少为 30 RPM；
3. 编码器实际速度低于配置的堵转 RPM；
4. 上述条件持续达到配置时间。

任一轮满足后设置 `FAULT_MOTOR_STALL`，目标清零并锁存。它是根据控制努力和编码器反馈
推断的“软件堵转”，不是电流保护。若需要检测短路或驱动器过流，应增加电流传感器。

## 4. 差速里程计

每个控制周期先把左右编码器增量换算成轮子位移：

```text
每计数距离 = π × 轮径 / 编码器每圈计数
左距离 dl = 左计数增量 × 每计数距离
右距离 dr = 右计数增量 × 每计数距离
中心距离 dc = (dl + dr) / 2
航向增量 dθ = (dr - dl) / 轮距
```

再使用周期中点航向累计坐标：

```text
x = x + dc × cos(θ + dθ/2)
y = y + dc × sin(θ + dθ/2)
θ = θ + dθ
```

航向角不限制在 ±180°，这样连续旋转时 `TURN` 不会遇到跨界问题。

## 5. 定距和定角

`MOVE` 使用左右累计距离平均值判断进度；`TURN` 使用里程计航向变化判断进度。接近目标时
按剩余量降低目标 RPM，最终进入 MANUAL 并清零目标。实际停止误差还受加速度、惯性、
地面摩擦和参数标定影响。

正的 `TURN` 默认代表左转：左轮后退、右轮前进。若实际方向相反，应先修正编码器方向和
电机接线，不要简单把角度符号“凑正确”。

## 6. 路线队列

路线位于 RAM，最多 8 段。`QMOVE/QTURN` 仅加入队列，不立即运动；`RUN` 从第 1 段执行。
每段完成后 BehaviorTask 自动装载下一段。急停、低电或堵转会取消本次运行，但保留已编辑
的路线，故障解除后可重新 `RUN`。

## 7. 运行诊断

`DIAG` 输出：

- 当前 FreeRTOS 剩余堆；
- 启动以来最小剩余堆；
- 七个任务的栈高水位，单位为 32 位字；
- RCC 复位原因位图。

栈高水位越小，说明该任务越接近栈耗尽。实车覆盖所有功能后仍应保留至少几十个字的
余量；若接近零，应先分析局部数组和 `snprintf`，再有依据地增加任务栈。
